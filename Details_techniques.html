Organisation des répertoires

Le dossier principal peut porter le nom du réseau. Il comprend un seul fichier, showMap.html, qui appelle la page de la carte et deux sous-dossiers : network_map et network_data.
Le dossier network_map est identique pour tous les réseaux. On y trouve la page de la carte (Node_network_map.html), quelques fichiers annexes et un sous-dossier leaflet-src comprenant toutes les sources du moteur de rendu Leaflet.
Le dossier network_data contient les données du réseau, configuration et données géographiques, dans un cadre défini ci-dessous.

Dossier network_data

Structure des fichiers

En javascript, on ne peut pas lire des fichiers locaux directement depuis une page html (rejet requête CORS...). L'astuce pour contourner cette difficulté est de lire ces fichiers au chargement de la page en tant que script de Javascript.
Les données géographiques sont caractérisées par des noms de variable varname détaillés ci-dessous.
Ces données sont obtenues automatiquement par une requête Overpass sous forme de fichier GeoJson. Il suffit d'éditer chaque fichier pour rajouter une première ligne
varname =
sans rien modifier d'autre et de le sauver dans le dossier network_data avec le nom varname.txt
